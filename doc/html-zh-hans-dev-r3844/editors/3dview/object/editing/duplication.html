副本¶

有两种类型的物体复制操作，复制 以及引用物体数据的 关联复制。

复制¶

参考

模式: 编辑和物体模式
面板: 工具栏 ‣ 工具‣ 编辑‣ 复制
菜单: 物体‣复制物体
快捷键: Shift-D

这将创建所选物体的外观一致的副本。在与原始物体相同的位置创建副本，并自动进入 拖拽 模式。见下文示例。

该副本是一个新物体，与原始物体共享一些(默认为，所有材质，纹理和函数曲线)数据块，但是其他数据块是复制的，例如网格。这就是为什么这种复制形式有时被称为“浅关联”，因为并不是所有的数据块都是共享的; 其中一些是“硬拷贝”！

Tip

可以在 用户设置，选择复制操作时关联或复制哪些类型的数据块。

范例¶

正在编辑 Cone.002 物体的 Cone.006 网格。网格的唯一数据块ID名称如图中大纲视图高亮显示部分。

中间的锥体已经 (1)关联复制到左侧，(2)复制到右侧。

  • 正在编辑复制的右锥体，中间的原始锥体保持不变。网格数据已经被复制，而非关联。
  • 同样，如果在物体模式下编辑了右侧的锥体，原始锥体将保持不变。新物体的变换属性或数据块是新的副本，而非关联。
  • 复制出右侧锥体后，它继承了中间椎体的材质。材料属性保持关联，不进行复制。

如果要单独复制通常关联数据块，请参见上文。

关联复制¶

参考

模式: 物体模式
面板: 工具栏 ‣ 工具‣ 编辑‣ 关联复制
菜单: 物体‣ 关联复制
快捷键: Alt-D

用户还可以选择 关联复制 而不是 复制 ; 这被称为深度关联。这将创建一个新物体，其 所有 数据都关联到原始物体上。如果在 编辑模式 下修改其中一个关联的物体，则所有关联的副本都将被修改。变换属性（对象数据块）仍为副本，而非关联，因此，仍可以随意旋转，缩放和移动，且不会影响其他副本。深入讨论参考下文 复制范例。

关联
在 复制物体 操作面板中，与复制不同的是， 关联 复选框已经勾选。

Hint

如果要单独修改新关联的副本物体，必须 LMB 单击“属性”编辑器的 物体数据 面板中的编号，手动将对象转换为“单用户”副本。 (参见 数据块菜单.)

See also

数据独立 用于取消关联数据块。

范例¶

对象 Cone.001 为关联复制。虽然这两个锥体都是名称不一样的单独物体，但是在大纲视图可以看到，它们其实是共用同一个名为Cone的网格的。

左侧椎体是中间椎体的 关联复制 (使用 Alt-D)。

  • 在 编辑模式 中移动一个物体中的一个顶点，原始锥中的同一顶点也将被移动。网格数据是关联的，而非副本。
  • 相反，如果在物体模式下旋转或重新缩放其中一个椎体，另一个将保持不变。变换属性被直接复制，而非关联。
  • 与前面的例子一样，新生成的圆锥体继承了原始锥体的材料。材料属性是关联的，而非复制。

一个普通的桌子有一个顶部和四条腿。对一条对进行建模，然后关联复制出另外三条腿。如果今后修改网格，所有的桌腿仍然是一样的。关联复制也适用于一套酒杯，汽车上的车轮...任何有重复或对称性的情形。

See also

关联库复制

关联库 也是一种复制的形式。任何其他blend文件中的物体或数据块可以在当前文件中重复使用。

Hint

如果要将变换属性 (即对象数据块) 为"关联", 请参阅 父级 页面.
