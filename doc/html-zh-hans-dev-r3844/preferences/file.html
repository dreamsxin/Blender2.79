文件¶

用户设置 的 文件 标签页用于配置自动保存路径，以及blend文件，渲染图像等的默认路径。

文件路径¶

各种各样的外部文件可以通过以下的选项来定位。

字体
查找字体的默认路径。
纹理
查找图像纹理的默认路径。
渲染输出
渲染完成的图片/视频的保存位置。
脚本
查找Python脚本的额外路径，见下文 脚本路径。
声音
查找音频的默认路径。
临时文件夹
临时文件的储存位置。
Render Cache
快速缓存渲染图像的存储位置。
国际化(i18n)分支
本地svn翻译文件副本 /branches 的路径，用于在UI中直接翻译。
图像编辑器
打开外部图像编辑软件的路径.
动画播放器
外部视频播放器路径，用于播放渲染好的动画。

Note

这些路径全部需要手动配置， 不会 自动创建。

脚本路径¶

默认情况下，Blender会查找多个路径(与操作系统有关)。设置了用户脚本路径，Blender就会多查找一个路径。这个功能可用来储存脚本/模板/预设，使其独立于当前Blender的版本。

在指定的文件夹下，创建指定的子文件夹，这样Blender才能找到对应的文件。新建的文件夹结构必须与blender安装路径中的脚本路径相一致：

  • 脚本
  • 插件
  • 样式
  • 预设
  • 摄像机
  • 布料
  • 用户界面主题
  • 操作项
  • 渲染
  • ...
  • 设置启动文件
  • 模板中不是所有文件夹都需要预设

Warning

确保当前用户有访问指定的文件夹下可执行文件的权限。以MS-Windows为例，如果对可执行文件启用 以管理员权限运行 选项，会导致由于系统UAC限制，打开编辑器失败。提升程序的运行权限还存在潜在的风险。

自动执行:¶

处于安全因素考虑，Python脚本(包括驱动表达式)默认不被执行。

自动运行 Python 脚本
忽视安全因素，允许自动运行Python脚本。
排除的路径
在这些文件夹下的blend文件将 无法 自动运行Python脚本，用于指定不受信任来源的blend文件存储路径。

See also

Python安全

保存和读取¶

相对路径
系统默认使用 相对路径。
压缩文件

保存时压缩blend文件。

保存 .blend文件时压缩文件。密集的网格、大量打包的贴图或者场景中的大量元素都会导致blend文件变得很大。

该选项会降低Blender的关闭时，或者自动保存备份文件时的速度。使用该选项等于是用CPU运行时间交换文件大小。

加载用户界面
默认勾选，打开文件时，从文件加载窗口布局( 屏幕布局)。从 文件浏览器 打开文件时，可以在 打开Blender文件 面板单独修改该选项。
过滤文件格式

启用该选项，文件浏览器对话框中将只显示匹配的文件类型(比如，加载完整Blender设置时，只显示blend文件)。显示的文件类型可以修改。

扩展名过滤。

隐藏点文件/数据块
隐藏文件名以 . 开头的文件（linux和苹果系统下的 . 文件是隐藏的）
隐藏最近查看的路径
隐藏 文件浏览器 中 最近打开的文件 面板。
隐藏系统书签
隐藏 文件浏览器 中的系统书签。
显示缩略图
打开 文件浏览器 时，显示图片及视频的缩略图。
保存版本

为同一文件保存的版本数量(作为备份)。

该选项用于告知Blender，在手动保存文件时，在当前工作目录为文件保存指定数量的版本。这些文件的扩展名为 .blend1, .blend2 ，以此类推。越老的版本，末尾的数字越大。比如，默认设置为2，将保存3个版本的文件： *.blend (最后保存的), *.blend1 (前一次保存的) 和 *.blend2 (倒数第三次保存的)。

最近打开的文件
文件 ‣ 打开近期文件 显示的文件数量。
保存预览图像
文件浏览器 会根据需要生成图片和材质的预览。勾选此项可以将这些预览图保存到blend文件中(代价是增大文件体积)。

自动保存¶

保存会话
在退出Blender后总是保存blend文件，并在再次启动Blender时重新加载该文件。
自动保存临时文件

激活自动保存(创建临时文件)。

勾选此项，Blender会 自动 为你在临时文件夹(根据 用户设置 中 文件 标签页设置的路径)保存在制文件的副本。

自动保存的文件使用随机数字命名，并使用blend作为扩展名。

计时器

自动保存之间的时间间隔。

用于指定自动保存的时间间隔，单位为分钟。Blender安装版的默认值为5(5分钟)。最小值为1，最大值为60(每隔一小时保存一次)。

点此阅读更多关于自动保存选项。

文本编辑器¶

制表符自动转换为空格
自动将键入的 Tab 转换为空格键。
作者
当格式支持此特性时，为导出的文件添加作者名。
