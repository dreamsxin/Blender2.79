简介¶

每个 .blend 文件包含了一个数据库。数据库包含了文件中的场景，物体，网格，纹理等等。

一个文件可能包含多个场景，每个场景可能包含多个物体。物体可能包含多个材质，材质可能包含多张纹理。在不同的物体之间还可以创建链接关系。

大纲视图¶

大纲视图 编辑器显示了 .blend 文件中你所有的数据，用户可以很方便地在其中检视文件内容。

大纲视图 允许用户对物体做一些简单操作，比如选择，重命名，删除，链接，以及父子化。

阅读更多关于大纲视图

打包和解包数据¶

Blender能够将不同类型的数据封装(整合)进 .blend 文件中。举例来说，一个以 .jpg为后缀名的贴图可以通过 File ‣ External Data ‣ Pack into blend-file 放入 .blend 文件中。当 .blend 文件被保存时，这个图片文件的副本就会被放入 .blend 文件里面。这个图片文件的副本可以随 .blend复制或邮件寄到任何地方。

当你在标题栏看到一个小小的 “圣诞礼物盒”，那就说明该图片已经打包进 .blend 文件了。

解包数据¶

当你收到一个打包过的文件，你可以通过 File ‣ External Data ‣ Unpack into Files... 解包。文件打包后，同样可以回溯到原始路径，可以是相对路径或绝对路径(在解包到原始位置时需要)。

选项¶

使用当前目录中的文件(必要时创建)
将所有文件解包至与blend文件相同目录 // ，并将其分组放置在对应文件夹(如 ''textures'')。然而，如果目标文件已存在，Blender会使用该文件，而不是解包它。
将文件写至当前目录(覆盖现有文件)
将所有文件解包至与blend文件相同目录，并将其分组放置在对应文件夹(如 ''textures'')。如果目标文件已存在，Blender会覆盖该文件。
使用原始目录中的文件(必要时创建)
将所有文件解包至原始位置。然而，如果目标文件已存在，Blender会使用该文件，而不是解包它。
将文件写至原目录(覆盖现有文件)
将所有文件解包至原始位置。如果目标文件已存在，Blender会覆盖该文件。
禁用自动解包，保留所有已解包文件
取消操作，并停用 自动打包到 .blend文件 选项。
