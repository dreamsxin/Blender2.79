屏幕布局¶

屏幕布局下拉菜单

屏幕布局 是指预定义窗口布局。Blender的区块很灵活，用户可以针对不同的任务，如建模、动画、和脚本，自定义工作环境。常用在同一文件中不同工作环境的快速切换。关于如何移动边框、分割与合并区块见 区块控件 。

屏幕布局下拉菜单位于 信息编辑器 的标题栏，用户可以用来选择布局。

控件¶

屏幕布局
可用屏幕布局列表。见 默认屏幕布局 。
添加 +
单击 添加 按钮，会基于当前布局新建一个新的布局。
删除 X
用户可以使用 删除 按钮删除选中的屏幕布局。

Hint

默认每个屏幕布局都会 记忆 最后使用的 场景 。选中屏幕布局将切换布局，并跳转到该场景。

快捷键¶

使用 Ctrl-Right 与 Ctrl-Left 循环切换屏幕布局。

Note

在macOS用户可能需要在电脑的首选项中关闭 "Mission Control" 的快捷键，该选项位于 System Preferences ‣ Keyboard ‣ Shortcuts 。

默认屏幕布局¶

3D View Full:全屏的3D视图，用于预览场景
Animation:制作角色以及移动物体、改变形状和颜色等等。
Compositing:合成场景的不同部分(如背景, 角色, 特效)以及滤镜 (如颜色校正) 。
Default:Blender新文件的默认布局.。用于新建模型。
Game Logic:Blender游戏的规划和编程。
Motion Tracking:
 用于与视频编辑器一起做运动追踪。
Scripting:记录工作内容和/或写自定义脚本自动化Blender。
UV Editing:将网格物体展开为2D，以控制表面贴图映射。
Video Editing:剪切和编辑动画序列。

保存与覆盖¶

blend文件会保存屏幕布局。打开文件时，在文件浏览器启用 加载用户界面 ，文件打开后，Blender会使用文件的屏幕布局覆盖当前布局。见 加载用户界面 。

自定义屏幕布局可以保存为 启动文件 的一部分。

附加布局¶

Blender使用经验丰富的用户，可以考虑添加其它的屏幕布局，以适应自己的工作流程，提高工作效率效率。下面是一些例子：

建模:3D视图(顶, 前, 边和透视图)，属性编辑器用于编辑。
灯光:3D视图用于移动灯光，UV/图像编辑器用于显示渲染结果， 属性编辑器用于渲染和灯光属性以及其他控制。
材质:属性编辑器用于设置材质, 3D 视图用于选择物体，大纲视图，脚本库(如果需要的话)，节点编辑器(如果用到了 基于节点的材质)。
绘制:UV/图像编辑器用于纹理图像绘制，3D视图用于在UV面选择模式直接在物体表面绘制物理，底部3个迷你3D视图用于显示参考背景图片，属性编辑器。
